2. 別再用 Float 了！現代定位雙雄：Flex 與 Grid
你說的 float 是 15 年前為了讓文字環繞圖片用的，拿來做版面簡直是災難。現在全世界只用這兩個：
---------------------------------------------------------------------------------------------------------
A. Flexbox (彈性佈局) —— 處理「一維」排列 (最常用)
想像你在排隊。你可以決定隊伍要橫著排（Row）還是直著排（Column），人與人之間要靠左、置中、還是平均分配空間。

Vaadin 對應：VerticalLayout 本質就是 flex-direction: column；HorizontalLayout 就是 flex-direction: row。

核心屬性：

justify-content: 主軸對齊（左右）。

align-items: 交叉軸對齊（上下）。
---------------------------------------------------------------------------------------------------------
B. CSS Grid (網格佈局) —— 處理「二維」佈局
想像你在畫 Excel 表格。你可以定義有幾欄、幾列，甚至讓一個組件佔據「兩欄三列」。
這對於複雜的 Dashboard 非常有用。

3. 單位之爭：px, %, em, rem 到底怎麼選？
你覺得解析度不同會亂掉，就是因為用了太多 「絕對單位 (px)」。
單位性質使用建議
---------------------------------------------------------------------------------------------------------
px絕對值禁止 用在寬度或大容器。只用於「邊框 (1px)」或「微小的固定間距」。
---------------------------------------------------------------------------------------------------------
%相對父層用於簡單的比例（如：左側選單佔 20%）。
---------------------------------------------------------------------------------------------------------
rem相對根字體最推薦。
通常 1rem = 16px。如果你把所有間距和字體改用 rem，
當使用者縮放瀏覽器字體時，整個版面會一起等比例縮放。
---------------------------------------------------------------------------------------------------------
vw / vh相對螢幕寬高100vw = 整個螢幕寬。適合做「全螢幕」的背景或區塊。
---------------------------------------------------------------------------------------------------------

4. Margin vs Padding：一個圖解就懂
這是新手最常搞混的地方。請記住 「盒子模型 (Box Model)」：
---------------------------------------------------------------------------------------------------------
Padding (內襯)：盒子裡面的填充。如果你想讓文字離邊框遠一點，用 Padding。它會影響盒子的大小。
---------------------------------------------------------------------------------------------------------
Margin (外距)：盒子與盒子之間的距離。如果你想讓兩個按鈕不要黏在一起，用 Margin。
---------------------------------------------------------------------------------------------------------
專業小撇步 (Gap)：現代 CSS 有一個屬性叫 gap。
如果你用 Flexbox 或 Grid，直接在父層設 gap: 1rem，
子元素之間就會自動產生間距，你連 Margin 都不用寫！
這在 Vaadin 的 Layout 中也有對應的 setSpacing(true)。
---------------------------------------------------------------------------------------------------------

5. 後端人的「切版救星」：Tailwind CSS 思維
如果你不擅長寫複雜的 CSS 檔案，我強烈建議你去參考 Tailwind CSS 的概念。 它把 CSS 拆成一堆小類別（Utility Classes），例如：

m-4 (margin 4)

p-2 (padding 2)

flex (啟動彈性佈局)

w-full (寬度 100%)

在 Vaadin 24+ 中，你可以直接使用這些 Lumo Utility Classes。
